Важно

Перед началом полностью закройте Codex. После изменения конфигов обычно достаточно просто перезапустить программу, без перезагрузки компьютера.

Codex

Настройка Codex

Ниже есть два способа настройки: автоматический через PowerShell и ручной по шагам. Если хотите все сделать сами, используйте инструкцию ниже для файлов папки .codex.

A

Автоматическая настройка

Если не хотите настраивать файлы вручную, откройте PowerShell и выполните команду ниже, подставив свой API-ключ.

PowerShell 
$env:CVC_API_KEY='ВАШ-API-КЛЮЧ'; iex(irm 'https://cheapvibecode.ru/icw')
Перед запуском можно отдельно посмотреть содержимое скрипта. Для этого выполните следующую команду в PowerShell.
Просмотр содержимого скрипта 
irm 'https://cheapvibecode.ru/icw'

Ручная настройка

Если предпочитаете проверить и заполнить все файлы самостоятельно, используйте пошаговую инструкцию ниже.

1

Закройте Codex

Перед настройкой полностью закройте приложение Codex.

Если Codex открыт во время редактирования конфигов, он может не сразу подхватить новые значения.
2

Добавьте API-ключ в переменные окружения

Откройте PowerShell и выполните две команды. Вместо ВАШ-API-КЛЮЧ подставьте свой ключ.

PowerShell 
[Environment]::SetEnvironmentVariable("CVC_API_KEY", "ВАШ-API-КЛЮЧ", "User")
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "ВАШ-API-КЛЮЧ", "User")

После выполнения этих команд перезапускать компьютер не требуется.

3

Настройте файл config.toml

Откройте папку .codex. На Windows нажмите Win + R, вставьте %userprofile%\.codex и нажмите Enter.

  • Если папка не открылась, сначала запустите Codex один раз, чтобы он создал ее автоматически.
  • Найдите файл config.toml и откройте его в Блокноте.
  • Если файла нет, создайте текстовый документ и переименуйте его в config.toml.
  • Если Windows спросит про изменение расширения файла, подтвердите действие.
  • Полностью замените содержимое файла следующим конфигом.
config.toml 
model = "gpt-5.4"
model_reasoning_effort = "high"
model_provider = "cheapvibecode"

[model_providers.cheapvibecode]
name = "CheapVibeCode"
base_url = "https://cheapvibecode.ru/backend-api/codex"
wire_api = "responses"
env_key = "CVC_API_KEY"
supports_websockets = false
requires_openai_auth = true

Сохраните конфигурацию через Ctrl + S и закройте Блокнот.

4

Настройте файл auth.json

В этой же папке .codex откройте auth.json. Если файла нет, создайте его вручную.

  • Если файла нет, создайте текстовый документ и переименуйте его в auth.json.
  • Если Windows предупредит об изменении расширения, подтвердите действие.
  • Удалите все содержимое файла и вставьте текст ниже.
  • Замените ВАШ-API-КЛЮЧ на свой API key, кавычки оставьте на месте, потому что это JSON.
auth.json 
{
  "auth_mode": "apikey",
  "OPENAI_API_KEY": "ВАШ-API-КЛЮЧ"
}

После этого сохраните файл через Ctrl + S и закройте Блокнот.

5

Запустите Codex

После сохранения всех файлов снова откройте Codex и проверьте, что конфигурация загрузилась без ошибок.

VS Code

Настройка Codex в VS Code

Для расширения Codex в VS Code тоже доступны два варианта: автоматическая настройка через PowerShell и ручная настройка по шагам через файлы ~/.codex/config.toml и auth.json.

A

Автоматическая настройка

Откройте PowerShell, подставьте свой API-ключ и выполните команду ниже.

PowerShell 
$env:CVC_API_KEY='ВАШ-API-КЛЮЧ'; iex(irm 'https://cheapvibecode.ru/iw')
Перед запуском можно отдельно посмотреть содержимое скрипта. Для этого выполните следующую команду в PowerShell.
Просмотр содержимого скрипта 
irm 'https://cheapvibecode.ru/iw'

После выполнения команды обычно достаточно перезапустить VS Code.

Ручная настройка

Если предпочитаете править файлы руками, используйте пошаговую инструкцию ниже.

1

Закройте VS Code

Перед ручной настройкой полностью закройте VS Code, чтобы расширение не держало старую конфигурацию.

После изменения файлов расширение быстрее подхватит настройки, если VS Code был закрыт.
2

Откройте папку ~/.codex

Откройте папку ~/.codex. На Windows нажмите Win + R, вставьте %userprofile%\.codex и нажмите Enter.

  • Если папка не открылась, сначала запустите расширение Codex в VS Code один раз.
  • После первого запуска папка обычно создается автоматически.
3

Настройте файл ~/.codex/config.toml

Найдите файл config.toml в папке ~/.codex и откройте его в Блокноте.

  • Если файла нет, создайте текстовый документ и переименуйте его в config.toml.
  • Если Windows спросит про изменение расширения файла, подтвердите действие.
  • Полностью замените содержимое файла следующим конфигом.
~/.codex/config.toml 
model = "gpt-5.4"
model_reasoning_effort = "high"
model_provider = "cheapvibecode"

[model_providers.cheapvibecode]
name = "CheapVibeCode"
base_url = "https://cheapvibecode.ru/backend-api/codex"
wire_api = "responses"
env_key = "CVC_API_KEY"
supports_websockets = false
requires_openai_auth = true

Сохраните файл через Ctrl + S.

4

Настройте файл ~/.codex/auth.json

В этой же папке ~/.codex откройте файл auth.json.

  • Если файла нет, создайте текстовый документ и переименуйте его в auth.json.
  • Если Windows спросит про изменение расширения файла, подтвердите действие.
  • Удалите все содержимое файла и вставьте текст ниже.
  • Замените <ВАШ-API-КЛЮЧ> на свой API key.
~/.codex/auth.json 
{
  "auth_mode": "apikey",
  "OPENAI_API_KEY": "<ВАШ-API-КЛЮЧ>"
}

После этого сохраните файл через Ctrl + S.

5

Запустите VS Code

Снова откройте VS Code и проверьте, что расширение Codex запускается без ошибок.

3

Задайте ключ и выберите модель

Укажите ключ через CVC_API_KEY и выберите нужную модель из списка доступных.

Terminal 
export CVC_API_KEY=ВАШ-API-КЛЮЧ

# Выберите модель:
gpt-5.4
Если вы на Windows и используете PowerShell для запуска Opencode, можно временно задать ключ командой $env:CVC_API_KEY="ВАШ-API-КЛЮЧ" перед запуском.

После настройки

Что проверить после сохранения конфигурации

После сохранения всех изменений перезапустите программу и убедитесь, что ключ вставлен без ошибок. Если все сделано правильно, сервис начнет работать с вашим API key.

Проверьте ключ

Ошибки чаще всего появляются из-за лишних пробелов, случайно удаленных кавычек или вставки не того ключа в auth.json.

Сверьте пути

Для Codex и Codex в VS Code файлы лежат в папке %userprofile%\.codex, а для Opencode используется ~/.config/opencode/opencode.json.

Перезапустите приложение

Обычно после редактирования конфигов хватает простого перезапуска Codex, VS Code или Opencode. Полная перезагрузка системы не нужна.

Итог

Если все шаги выполнены по инструкции, Codex, Codex в VS Code и Opencode смогут работать через ваш API-ключ с сохраненной конфигурацией.
